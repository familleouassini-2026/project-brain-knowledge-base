**Document ID:** FS-003  
**Version:** 1.0.0  
**Status:** Draft  
**Owner:** Project Brain  
**Last Update:** 2026-07-03

## Purpose

This document defines the non-negotiable rules that guide Project Brain.

## Principles

1. Documentation is the source of truth.
2. Code is an implementation, not the main asset.
3. AI proposes; humans decide.
4. No major architectural change without impact analysis.
5. Every important decision must be traceable.
6. The smallest compatible change is preferred.
7. The system must survive its authors.
8. Every feature must be explainable.
9. Every bug must be traceable to a rule, specification or implementation gap.
10. Simplicity is an architectural requirement.
