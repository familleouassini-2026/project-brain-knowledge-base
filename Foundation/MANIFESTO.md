## Purpose

This document expresses the identity and commitments of Project Brain.

## Manifesto

We do not build software only to make it work today.

We build it so that it can still be understood tomorrow.

We believe that the first responsibility of a complex project is to preserve its own meaning.

We value:
- clarity over cleverness
- documentation over memory
- traceability over improvisation
- explainability over blind automation
- controlled evolution over uncontrolled acceleration
